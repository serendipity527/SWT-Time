# 使用 GPU 1 启动 Time-LLM 项目

## 前提条件检查

### 1. 确认环境已激活
```bash
conda activate timellm
```

### 2. 确认数据集已下载
```bash

# 应该看到：ETTh1.csv, ETTh2.csv, ETTm1.csv, ETTm2.csv
```

## 方案 3：修改官方脚本使用 GPU 1

### 创建自定义脚本

创建文件 `scripts/TimeLLM_ETTh1_GPU1.sh`：

```bash
#!/bin/bash

model_name=TimeLLM
train_epochs=10
learning_rate=0.01
llama_layers=32
batch_size=8
d_model=32
d_ff=128

comment='TimeLLM-ETTh1-GPU1'

# 使用 GPU 1
export CUDA_VISIBLE_DEVICES=1

python run_main.py \
  --task_name long_term_forecast \
  --is_training 1 \
  --root_path ./dataset/ETT-small/ \
  --data_path ETTh1.csv \
  --model_id ETTh1_512_96 \
  --model $model_name \
  --data ETTh1 \
  --features M \
  --seq_len 512 \
  --label_len 48 \
  --pred_len 96 \
  --factor 3 \
  --enc_in 7 \
  --dec_in 7 \
  --c_out 7 \
  --des 'Exp' \
  --itr 1 \
  --d_model $d_model \
  --d_ff $d_ff \
  --batch_size $batch_size \
  --learning_rate $learning_rate \
  --llm_layers $llama_layers \
  --train_epochs $train_epochs \
  --model_comment $comment
```

**运行**：
```bash
chmod +x scripts/TimeLLM_ETTh1_GPU1.sh
bash scripts/TimeLLM_ETTh1_GPU1.sh
```

## 常见问题

### 问题1：CUDA out of memory
**解决**：减小 `--batch_size`（8 → 4 → 2）

### 问题2：模型下载失败
**解决**：
```bash
export HF_ENDPOINT=https://hf-mirror.com
```

### 问题3：找不到数据集
**解决**：确认数据集路径和文件名正确
